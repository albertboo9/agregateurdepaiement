Étapes d'Intégration

1. Créer le dossier du provider

Crée un nouveau dossier dans apps/backend/providers/ avec le nom du provider.

mkdir apps/backend/providers/nouveau_provider

2. Créer le fichier de service

Crée nouveau_provider.service.js qui étend PaymentProviderInterface.

import { PaymentProviderInterface } from "../base/provider.interface.js";
import { PaymentStatus } from "../../enums/index.js";

export class NouveauProviderService extends PaymentProviderInterface {
    constructor(config = {}) {
        super();
        this.apiKey = config.apiKey || process.env.NOUVEAU_PROVIDER_API_KEY;
    }
    
    async createPayment(paymentData) { /* ... */ }
    async checkStatus(transactionNumber) { /* ... */ }
}

3. Implémenter les méthodes obligatoires

  • createPayment(paymentData) - Crée un paiement
  • checkStatus(transactionNumber) - Vérifie le statut
  • validateWebhookSignature(payload, signature) - Valide les webhooks
  • mapStatus(providerStatus) - Convertit le statut

4. Ajouter au ProviderFactory

Modifie apps/backend/providers/index.js:

import { NouveauProviderService } from "./nouveau_provider/nouveau_provider.service.js";

case ProviderCode.NOUVEAU_PROVIDER:
    return new NouveauProviderService(config);

5. Ajouter les routes webhook

Modifie apps/backend/routes/webhook.routes.js:

router.post("/nouveauprovider", catchAsync(WebhookController.nouveauprovider));

6. Ajouter le controller webhook

Modifie apps/backend/controllers/webhook.controller.js:

static async nouveauxprovider(req, res) {
    const signature = req.headers['x-signature'];
    await WebhookProcessor.processEvent('nouveauprovider', req.body, signature);
    res.status(HttpStatus.OK).json({ received: true });
}

7. Ajouter au webhook processor

Modifie apps/backend/services/webhook-processor.service.js:

// Identifier la transaction
else if (providerCode === "nouveauprovider") {
    transactionNumber = payload.reference || payload.transaction_id;
}

// Mapper le succès
if (provider === "nouveauprovider") {
    return payload.status === "success";
}

8. Ajouter les variables d'environnement

Modifie apps/backend/.env:

NOUVEAU_PROVIDER_API_KEY=ta_cle_api
NOUVEAU_PROVIDER_SECRET_KEY=ta_cle_secrete
NOUVEAU_PROVIDER_SANDBOX=true

9. Ajouter au seed

Modifie apps/backend/scripts/seed-providers.js:

// Provider
{
    code: "NOUVEAU_PROVIDER",
    name: "Nouveau Provider",
    isActive: true,
    supportCard: true,
    supportMobileMoney: true
}

// Route
{
    providerId: createdProviders["NOUVEAU_PROVIDER"].id,
    countryCode: "BJ",
    currency: "XOF",
    priority: 1
}

10. Tester l'intégration

Exécute le seed et teste:

cd apps/backend && node scripts/seed-providers.js

Checklist d'Intégration

  • Dossier provider créé
  • Service extends PaymentProviderInterface
  • Méthodes createPayment et checkStatus implémentées
  • Validation webhook implémentée
  • Mapping des statuts correct
  • Provider ajouté au Factory
  • Route webhook ajoutée
  • Controller webhook ajouté
  • Webhook processor mis à jour
  • Variables .env configurées
  • Provider ajouté au seed
  • Test local effectué
  • Déploiement en production
